CIRCULAR WEATHER

Purpose of this project

This project presents a 24-hour weather prognosis for 3 days. In the project we used data from an open weather API. The data is visualized through a circular diagram. Each day have 3 circles(1 for temperature 1 for humidity and 1 for wind levels) and each small circle represents an hour of the day. The user can choose a city of their choice. The purpose of this project was to experiment with data visualization while it was a theme everyone in the group found interesting. Data visualization finds practice a lot in architectural fields

CODE ANALYSIS

We use loadJSON(url) in order to turn our data into JSON format (making them into objects). With SelectPlace() and handleData(data) we locate the data we want to use.

In the draw function we draw the circles. We use if statement in nested loop in order to determine the difference in colors as different data progresses. Specifically the color changes for different temperatures but also as high as the temperature is so the size of the circle changes and becomes bigger. For the circles we use a mutual center x,y and we use sin and cos to draw the circles that surround it .
